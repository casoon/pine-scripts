## v0.7 — 2026-06-27
- **Two-tier alerts (⚠️ Watch → ✅ Signal)** — new early-warning tier fires when WT reaches its deep extreme (a reversal setup is forming), giving lead time on 1h/4h before the trigger cross. Shown on the PRICE chart (long below the bar, short above) via a selectable "Watch Marker Style" (Label ⚠ / Dot / Triangle / Background / Off), so the warning is visible, not alert-only. One combined `alert()` ("any alert() function call", once per bar close) now carries both tiers with full data (direction, grade/max, ★, type, ticker, TF, price), forced to bar close so it never repaints on intrabar HTF updates. Named alerts added for Watch Long/Short, plus the existing signal/high-grade alerts, all with `{{interval}}`/`{{close}}` placeholders.
- **Auto-shorten WT length above 4h** — over daily bars the base channel/average/signal lengths (10/21/4) make WaveTrend too slow and the crosses unusable (smoothing only adds lag). New toggle (default on) swaps in shorter lengths — "Channel Length (>4h)" / "Average Length (>4h)" / "Signal Length (>4h)", default 5/11/3 — automatically on Daily+ charts. Tunable.
- **Deep tier decoupled from exhaustion** — a cross out of a deep extreme (|WT| ≥ extreme level) now scores its depth (2) immediately, without waiting for the exhaustion pullback. Previously a real bottom/top reversal that had just printed its extreme graded 0 and was dropped (in one 1h sample, 306 deep crosses were suppressed by this alone). The exhaustion pullback now only gates the shallow zone tier.
- **Divergence adds to grade (+1)** — new toggle (default on): a present price/WT divergence now lifts the grade by +1 and marks the signal Reversal, instead of being a draw-only ★. This lets a counter-HTF reversal at the zone (depth 1 + divergence) reach the min grade and actually fire. Still never a gate. Debug candidate logs now include `div=Y/N`.
- **Horizon Mode (Fixed / Relative)** — new Relative mode scales Tide/Wave as multiples of the chart TF (defaults: Wave = chart × 4, Tide = chart × 24), so the Ripple < Wave < Tide nesting holds on every chart including 4h and Daily. Fixed mode (default) keeps the absolute Tide/Wave TFs as before.
- **All on-chart text in English** — phase names (Bottom/Up/Top/Down), label tooltips, table verdict/state, and the log are now fully English.
- **MTF grading (Auto / On / Off)** — new "MTF grading (Wave + Tide)" mode. Off = single-TF mode: the grade is the chart-TF depth only, Wave/Tide/Wave-Stoch dropped. **Auto (default)** keeps MTF on up to a 4h chart and turns it OFF above 4h (Daily+) in Fixed mode, where the nesting degenerates (Tide = chart, Wave = intraday noise — every signal got a free Tide point and inflated to G4). Relative mode keeps MTF on at every TF. The table flags "MTF auto-off (>4h) · depth only".
- **Trigger Type input (WT-Cross / WT-Turn)** — new WT-Turn option fires when WT1 itself turns (slope flip out of the extreme), ~1 bar earlier than the classic WT1×WT2 cross. In WT-Turn mode the turn is treated as the exhaustion event, so the depth grade is awarded at the turn instead of waiting for the extra pullback (which previously re-delayed it). Same zone, grade and cooldown apply to both; trigger dots in the pane follow the selected type.
- **Signal type** — every signal is now read as Reversal / Continuation / Mixed (deep cross vs. shallow-but-aligned cross), shown in the label tooltip and the log. Disambiguates what a given grade actually means.
- **Room read-out** — tooltip/log show whether Wave and Tide still have room to run; measured against the trigger zone (±zoneLevel), not the ±extreme — the extreme was almost never reached at signal time, so room used to read "yes" every time. Information only, never a filter.
- **Suppressed-trigger hints** — optional faint markers for a triggered cross that scored at least Grade 1 but stayed below the minimum (e.g. a 1h long crossing while 4h+1D are both down → counter-HTF, Grade 1). Direction-agnostic so counter-HTF suppression is visible, not hidden. New "Show Grade-1 Follow-Up Hints" toggle.
- **Last Bias** — the table's "Position" row is renamed and now shows just the last signal direction (an opposite signal can close/re-open on the same bar, so it was never a held position).
